# Design – Make Page Audio Splits & Slide Packages

## Overview
The goal of this change is to transform a single video (`A2Z-original.mp4`) and its Chinese subtitle file (`A2Zsrt.srt`) into a collection of per‑page assets that can be used for PPT animation.

### Asset Generation
1. **Audio Splitting** – Using the subtitle timestamps, each non‑empty subtitle block is cut from the extracted audio file (`A2Z-original-audio.mp3`). The resulting files are named `audio‑N.mp3` where `N` starts at 1.
2. **Slide Extraction** – Slide boundaries are detected via `ffmpeg` scene‑change detection (`gt(scene,0.4)`). The first slide is forced to start at **00:01** to satisfy the user requirement.
3. **Packaging** – For every page a folder `slide‑N/` is created containing:
   - `audio‑N.mp3`
   - `slide‑N.png` (image captured at the slide start timestamp)
   - `process_page.sh` – a tiny helper script that can regenerate the assets for that page.

### Directory Layout
```
user/assets/
├─ A2Z-original.mp4                # source video
├─ A2Z-original-audio.mp3          # extracted audio
├─ A2Zsrt.srt                      # Chinese subtitles
├─ split_pages.sh                  # main script (generated by this change)
└─ slide‑pages/
   ├─ slide‑1/
   │   ├─ audio‑1.mp3
   │   ├─ slide‑1.png
   │   └─ process_page.sh
   ├─ slide‑2/ …
   └─ …
```

### Key Constraints
- **Slide‑1 start time** must be exactly 1 second.
- **Silent subtitles** (empty text) are skipped – no folder/audio is created for them.
- All processing is performed with `ffmpeg -c copy` for audio (lossless) and PNG extraction for slides.
- The workflow is fully script‑driven; no manual editing of timestamps is required.

## Dependencies
- `ffmpeg` (required for both audio cutting and image extraction).
- `awk` (standard macOS utility) for parsing the SRT file.

## Failure Modes & Mitigations
- **Scene detection false positives/negatives** – The `THRESHOLD` variable can be tuned. If slide boundaries are off, the user can adjust the threshold in `split_pages.sh`.
- **Missing subtitles** – If a subtitle entry has no text, it is ignored, preventing empty audio files.
- **File path errors** – All paths are relative to `user/assets`; the script aborts on any missing input file.
